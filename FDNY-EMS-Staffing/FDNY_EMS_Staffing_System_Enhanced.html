      FDNY EMS Staffing System - User Guide

      Latest Enhancements - November 2025

      • Auto-Sync Every 30 Seconds: Real-time collaboration with automatic change detection
      • Change Notifications & Log: Popup alerts + permanent log in Exceptions tab
      • Time Prompts for Mutual In/TCI: Automatic time entry with validation
      • Multiple Shifts Support: Employees working 2 shifts appear twice in exports
      • Seniority Sort: Order by Original Date of Appointment (ODA)
      • Enhanced Overtime Calculation: Scans all data sources comprehensively
      • Combined Leadership Column: Space-saving 3-column layout in exports
      • Portrait-Optimized Printing: All exports fit letter paper perfectly
      • Row Numbering: Automatic numbering restarts per tour group
      • 20% Larger Export Fonts: Improved readability for printed reports

      Quick Start

      1. Login: Use admin account (username: admin, password: 1234567!) or your last name & shield number
      2. Load Employee Data: Pull roster from cloud OR upload CSV file via " Admin" "Data Management"
      3. View Today's Schedule: The main roster shows employees scheduled to work based on platoon rotation
      4. Mark Absences: Search for employees in the "Absent" panel and select a reason
      5. Add Overtime: Search in "Added to Schedule" panel to add employees working overtime
      6. Export Report: Click "Export HTML" to generate a printable daily rundown

      Main Features

      Employee Roster

      • Search: Use the search bar at the top to quickly find employees
      • Filter by Tour: Select T1, T2, or T3 to view specific tours
      • Filter by Function: Check boxes for CHIEF, CAPTAIN, SGT, RD, ARD, etc.
      • Working Today Filter: Toggle to show only employees scheduled today
      • Seniority Sort: Click "Sort by Seniority" to order by Original Date of Appointment (ODA)
        Gold banner appears when active: " Sorted by Seniority (ODA)"
        Sort order persists through filter changes
        Use "Clear All Filters" to reset
      • Hide Inactive: Toggle to hide INACTIVE status employees
      • Drag & Drop: Drag employees between Roster, Absent, and Added lists

      Absent Panel

      • Add to Absent: Search employee name or drag from roster
      • Select Reason: Choose from dropdown (SICK, ML, FMLA, etc.)
      • Time-Based Reasons: When selecting "Mutual In", "MI", "TCI", or "Tour Change In", system prompts for:
        Start time in HH:MM format (24-hour)
        Optional swap partner info (shield & name) for Mutual In
      • Overtime Option: Click "OT" to log overtime details (hours, vice, reason)
      • Remove: Click the "" button to remove from absent list

      Added to Schedule Panel

      • Add Employee: Search name or drag from roster (for off-duty employees working OT)
      • Enter Reason: Specify why they're added (e.g., "Filling in", "Voluntary OT")
      • Time Prompts: System automatically prompts for start time when selecting:
        "Mutual In" or "MI" - includes optional swap partner info
        "TCI" or "Tour Change In" - requires start time
      • Multiple Shifts: Employees working both regular shift + added shift appear twice in exports
        Original shift shows normally
        Second shift labeled as "(2nd Shift)" or "(2nd)"
      • Override Shield: Enter alternate shield number if needed
      • Override Tour: Change tour assignment if different from normal
      • Overtime: Click "OT" button to log full overtime details

      Platoon Rotation System

      The system automatically calculates which platoons are working based on rotation schedules:

      • A, B, C Platoons: 15-day rotation (Work 5, Off 2, Work 5, Off 3)
      • D Platoon: Monday-Friday (weekends off)
      • E, F, G, H Platoons: 14-day rotation cycle

      Working platoons display automatically at the top of the roster for any selected date.

      Calendar & Schedule Views

      Calendar View

      • Click " Calendar" button in the header
      • Navigate months using arrows
      • Click any date to view roster for that day
      • Color-coded days show absence/schedule density
      • Schedule absences in advance by clicking dates

      Schedule View (Week/Month)

      • Click " Schedule" button to open weekly schedule
      • View 7-day or full month grid of all employees
      • Mark absences for future dates by clicking employee rows
      • Toggle between Tours and Functions using filters
      • Schedule automatic absences (vacations, planned ML, etc.)
      • Click "OT/Switch" to log scheduled overtime or tour switches

      Export & Printing

      Four export layouts available:

      • Original Layout: Complete detailed rundown with all sections and statistics
        Portrait-optimized, includes highlighted employees (red=absent, green=added, orange=overtime)
        Shows row numbering, 3-column grid (Leadership/Supervisors combined)
        Enhanced overtime calculation from all sources
        Multiple shifts: Shows employees twice if working both regular shift + added shift
      • Compact Layout: Condensed format grouped by tour and function
        Portrait print-friendly, shows availability percentage
        Multiple shifts: Employees appear twice with "(2nd Shift)" label
      • Print-Optimized: Landscape (7 columns) or Portrait (6 columns) single-page format
        Filters out off-duty employees, shows only working staff
        Multiple shifts: Displays "(2nd)" row with start time
      • Cards Layout: Individual employee cards with full details

      To Export:

      1. Select layout from dropdown next to "Export HTML" button
      2. Click "Export HTML"
      3. New window opens with formatted report (all exports portrait-optimized for printing)
      4. Use browser's Print function (Ctrl+P) or Save as PDF

      Enhanced Features:

      Comprehensive overtime tracking from all data sources
      Combined Leadership/Supervisors column saves space
      Row numbering restarts for each tour group

      Statistics & Analytics

      View Statistics Page

      • Access: Click " View Statistics" button in the settings panel
      • Overall Summary: Total employees, working today, present, absent, off-duty, added counts
      • Enhanced Overtime Tracking: Comprehensive calculation from multiple sources
        Scans overtime tracking data
        Analyzes absence reasons for overtime entries
        Checks added-to-schedule reasons
        Prevents duplicate counting with intelligent deduplication
      • Platoon Status: See which platoons are working and staffing percentage
      • Tour Breakdown: Statistics by Tour 1, 2, and 3
      • Function Breakdown: Stats for each function (CHIEF, CAPTAIN, SGT, RD, ARD, etc.)
      • Leave Usage Statistics: Track sick days and other leave by employee
        Filter by 7, 30, 90 days, or 1 year periods
        Filter by Function, Tour, or Platoon
        Search by employee name
        Show only employees with leave usage
        Color-coded usage rates (green=low, orange=medium, red=high)

      Cloud Sync (Captains, Chiefs & Admins Only)

      Supabase Cloud Integration

      • Auto-Sync Schedule Data: Absences and schedule changes automatically sync every 30 seconds
      • Change Notifications: Popup banner shows when changes detected from other users
        Notification auto-dismisses after 15 seconds
        Click "" to dismiss manually
      • Change Log: Permanent log in "Exceptions" tab tracks all sync changes
        Green border = Employee added
        Orange border = Status/reason modified
        Individual "" buttons to remove log entries
        "Clear All" button to remove all logs
      • Sync Indicator: Shows sync status in header (animated when syncing)
      • Manual Roster Sync: Employee roster requires manual push/pull for safety
      • Push Roster to Cloud: Upload your current employee roster to cloud database
        Updates cloud with local changes, syncs across stations
      • Pull Roster from Cloud: Download employee roster from cloud database
        Replaces local data, automatically reloads all UI components
      • Station-Specific: Each station has its own isolated data
      • Data Preservation: Local roster data is preserved unless manually pulled from cloud
